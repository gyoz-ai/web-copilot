## Chrome Web Store

### Adding languages

1. Go to [Chrome Web Store Developer Dashboard](https://chrome.google.com/webstore/devconsole)
2. Select **gyoza** extension
3. Click **Store listing** in the left sidebar
4. At the top, click the **language dropdown** (shows "English" by default)
5. Click **Add language** and select the locale to add
6. Fill in the translated **name** (up to 75 chars) and **description** (up to 132 chars for short, full for detailed)
7. Upload localized **screenshots** if available (1280x800 or 640x400)
8. Click **Save draft**, then **Submit for review**

### Notes

- Chrome uses `_locales/` folder in the extension itself for manifest strings (name, description in toolbar). These are already generated by `bun run scripts/generate-locales.ts`.
- Store listing translations are **separate** from `_locales/` — they control what appears on the Web Store page itself.
- Chrome supports 50+ languages. Prioritize: English, Portuguese (BR), Spanish, French, German, Japanese, Korean, Chinese (Simplified/Traditional).
- Short description (132 chars max) appears in search results. Make it count.

## Firefox Add-ons (AMO)

### Adding languages

1. Go to [Firefox Add-on Developer Hub](https://addons.mozilla.org/developers/)
2. Click **Edit Listing** on the gyoza extension
3. Scroll to the **Description** section
4. Click the **locale switcher** (globe icon or dropdown at the top of the form)
5. Select **Add a new locale**
6. Choose the language and fill in:
   - **Name** (translated extension name)
   - **Summary** (short description, 250 chars max)
   - **Description** (full description, supports HTML)
7. Click **Save Changes**

### Notes

- Firefox also uses `_locales/` for manifest strings (same as Chrome, already generated).
- AMO auto-detects browser locale and shows the matching listing.
- Firefox supports fewer locales than Chrome. Check [AMO locale list](https://addons.mozilla.org/developers/docs/policies/content) for supported codes.
- Summaries are plain text. Descriptions support limited HTML (`<a>`, `<b>`, `<i>`, `<ul>`, `<li>`).

## Apple App Store Connect (Safari)

### Adding languages

1. Go to [App Store Connect](https://appstoreconnect.apple.com)
2. Select **gyoza** app
3. Navigate to **App Information** (left sidebar)
4. Under **Localizations**, click the **+** button
5. Select the language to add
6. Fill in:
   - **App Name** (30 chars max) — use "gyoza" for all languages
   - **Subtitle** (30 chars max) — short localized tagline
   - **Description** — full localized description
   - **Keywords** (100 chars, comma-separated) — localized search terms
   - **What's New** — localized changelog
7. Upload localized **screenshots** (required per device size: iPhone 6.7", iPad 13", Mac)
8. Click **Save**

### Notes

- Safari extension is distributed as a macOS/iOS app through the App Store.
- App-level localization (InfoPlist.strings) is handled by the build script (`bun run scripts/safari-localize.ts`), which runs automatically during `build:safari`.
- App Store Connect is the **only** place to manage store listing translations for Safari.
- Apple requires **screenshots for each locale and device class** before the localized listing goes live.
- Apple reviews take 1-3 days typically. Plan accordingly.

## Screenshot Tips

- Use the same set of pages/scenarios for all locales for consistency
- Set browser language before taking screenshots so UI strings match
- Key screenshots: (1) floating bubble on a real website, (2) chat interaction, (3) form filling in action, (4) settings/model selection
- Chrome: 1280x800 or 640x400
- Firefox: no strict size requirement, but 1280x800 recommended
- App Store: device-specific sizes required (use Xcode simulator)
